# STS2 Adviser UI 改进总结

## 已实现的改进

### 1. ✅ UI宽度和布局优化
- **窗口尺寸**: 340×420 → **480×560**（增加40%宽度、33%高度）
- **工具栏**: 改为三层布局
  - 第一层: [⟳ 评估] [🔄 检测] [⚙ 设置] 按钮
  - 第二层: 游戏信息显示（角色、楼层、HP、金币、卡组数）
  - 第三层: 三个指示灯水平排列
    - ● 后端 (后端连接状态)
    - ● 游戏 (游戏存档检测)
    - ● 日志 (日志监视状态)

### 2. ✅ 游戏状态实时检测
- **实时显示**: 当游戏运行时，自动显示：
  - 角色名称 (e.g., "SILENT")
  - 楼层数 (e.g., "F5")
  - 当前HP和最大HP (e.g., "65/70")
  - 金币数 (e.g., "150")
  - 卡组大小 (e.g., "12")
- **指示灯状态**:
  - 🟢 绿色: 游戏已运行且能检测到存档数据
  - 🔴 红色: 游戏未运行或存档未检测到

### 3. ✅ 中文卡牌名称支持
- **598个中文卡牌名称映射** 已加载
- **自动补全**:
  - 支持英文卡牌名称 (e.g., "Shiv", "Defend")
  - 支持中文卡牌名称 (e.g., "匕首", "防守")
- **智能转换**: 用户输入中文名称时自动转换为英文ID进行评估

### 4. ✅ 改进的错误处理
- **422 验证错误修复**: 当用户未提供游戏数据时，系统提供默认值
- **未找到卡牌时**: 显示清晰的错误消息
  - "❌ 未找到匹配的卡牌。请检查卡名是否正确（卡名需用英文）"

### 5. ✅ 配置管理
- **设置对话框 (⚙)**: 允许手动设置存档和日志路径
- **自动检测按钮 (🔄)**: 重新检测游戏和日志位置
- **路径持久化**: 配置保存到 ~/.sts2-adviser/config.json

## 使用说明

### 基本流程

1. **启动应用**
   ```bash
   python main.py
   ```

2. **等待游戏检测**
   - 后端启动（约2秒）
   - 日志监视启动
   - 如果游戏在运行，会在工具栏第二层显示游戏信息

3. **输入候选卡（三种方式）**
   - **中文名称**: 输入 "匕首, 防守, 力量"
   - **英文名称**: 输入 "Shiv, Defend, Power"
   - **混合输入**: 输入 "Shiv, 防守, Power"

4. **点击评估或按Enter**
   - 系统使用当前游戏状态进行评估
   - 如果游戏未运行，使用默认状态 (SILENT角色, 1楼)
   - 返回卡牌评分和推荐

### 指示灯说明

| 指示灯 | 绿色 | 红色 |
|--------|------|------|
| **后端** | FastAPI服务运行 | 服务无法连接 |
| **游戏** | 存档文件已读取 | 未检测到游戏 |
| **日志** | 日志文件正在监视 | 日志文件未找到 |

### 手动配置路径

如果自动检测失败：
1. 点击 ⚙ (设置) 按钮
2. 选择 "存档文件夹" (包含 current_run.save)
3. 选择 "日志文件夹" (包含 godot.log)
4. 点击 "保存"
5. 指示灯应变为绿色
